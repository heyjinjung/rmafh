# 문서 규칙

## 1. 공통 원칙
- 목적/결론을 먼저, 세부 근거는 뒤에 배치
- 간결한 문장과 리스트/표를 우선 사용
- 모든 문서 상단에 Changelog(버전, 날짜, 작성자, 변경 요약) 기입

## 2. 형식 규칙
- 파일명: 영어 대문자 스네이크 (예: README.md, ARCHITECTURE.md, DOC_RULES.md, DESIGN_VAULT_V2.md)
- 언어: 한국어 우선, 필요 시 영문 병기
- 마크다운 사용, 코드/쿼리/명령어는 코드블록으로 표기
- 용어 통일: LOCKED/UNLOCKED/CLAIMED/EXPIRED, “보관/소멸/회수 경고” 톤 유지

## 3. 문서 템플릿
- 헤더: 제목, 문서 타입, 버전, 작성일, 작성자, 대상 독자
- 요약: 3~5줄 핵심
- 본문: 배경 → 문제/목표 → 대안 → 결정 사항 → 영향도/리스크 → 후속 작업
- 부록: 용어, 참고 링크, 메트릭 정의

## 4. 버전/승인
- 버전: vX.Y.Z (Major=전략 변경, Minor=기능 추가, Patch=정오표/문구 조정)
- 변경은 PR로 제출, 요약/영향도/리뷰 포인트 명시
- 최소 1인 이상 리뷰 후 병합

## 5. 배치/관리
- 기본 위치: 리포지토리 루트 또는 docs/ 하위
- Deprecated 정보는 섹션으로 명시하고 제거 일정 기록
- 설계 ↔ 스키마 ↔ API ↔ UI ↔ 알림 문서 간 상호 링크 유지

## 6. 프로젝트별 추가 규칙 (Vault v2.0)
- 카피는 “보관/소멸/회수” 프레이밍, “보상/이벤트/충전” 표현 최소화
- 상태/진행률 표기는 금고별로 분리하여 명시 (골드/플래티넘/다이아)
- 만료/알림 시점은 UTC 표기 후 로컬 타임존 변환 규칙을 명기
